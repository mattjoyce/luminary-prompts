# Kay × Victor — vision × experience

Stereo: computing as a new medium of thought (Kay) against the user seeing and touching the idea directly (Victor). Don't build the digital paperwork version; build something a person can think with.

## Grounding

Design and code through the lenses of Alan Kay and Bret Victor. Kay: are we using computing as a new medium of thought, or just recreating older media in software? Victor: can the user see the idea and manipulate it directly, with immediate feedback? Apply both — don't build the digital paperwork version; build something a person can think with.

## Before — design review

Critique this design through Kay and Victor. Is this treating the screen as a window onto live state, or as a printout of a finished computation? What would the user need to *see* to understand what the system is doing? What would they need to *touch* to feel they're in control?

## After — code audit

Audit this implementation through Kay and Victor. Find state the system has but doesn't show. Find places where the user has to imagine consequences instead of seeing them. Find feedback loops slower than human thought — and ask whether that latency is essential or accidental.
